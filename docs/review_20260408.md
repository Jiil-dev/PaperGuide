# src/config.py 설계 (수정판 - 5가지 보정 + 1가지 결정 반영)

## 1. 모듈 책임 (한 줄)
**config.yaml을 PyYAML로 파싱하고, Pydantic v2로 검증하며, 경로를 자동으로 절대 경로로 변환 및 생성하여 완전히 타입이 안전한 설정 객체를 반환.**

## 2. 공개 인터페이스

```python
def load_config(path: str = "config.yaml") -> "Config":
    """
    config.yaml 로드, 검증, 및 경로 생성.
    
    동작:
    1. config.yaml의 절대 경로 구하기 (config_abs = Path(path).resolve())
    2. 프로젝트 루트 = config.yaml이 있는 디렉토리
    3. YAML 파싱 후 Pydantic 검증 (이 과정에서 
       YAML의 문자열 경로 → Pydantic이 자동으로 pathlib.Path로 변환)
    4. 상대 경로를 절대 경로로 정규화
    5. 설정 파일에 명시된 모든 디렉토리 자동 생성
       (단, data/cache/claude_responses/ 등 하위 폴더는 
        각 모듈이 필요할 때 자신의 책임 하에 생성)
    
    Args:
        path: config.yaml의 경로 (문자열). 기본값 "config.yaml".
              절대/상대 모두 가능. 상대 경로면 현재 스크립트 위치 기준.
    
    Returns:
        Config 인스턴스. config.paths의 모든 필드는 이미 pathlib.Path 타입이고,
        절대 경로로 정규화되어 있음.
    
    Raises:
        FileNotFoundError: config.yaml이 없을 때
        yaml.YAMLError: YAML 파싱 실패 시
        pydantic.ValidationError: 검증 실패 시 (필드 누락, 타입 오류, 
                                   오타가 있는 필드 등)
        OSError: 디렉토리 생성 권한 부족 등
    """

class Config(BaseModel):
    """전체 설정 모델"""
```

## 3. Pydantic v2 모델 구조 (수정판)

```python
from pydantic import BaseModel, Field, ConfigDict
from typing import Literal
from pathlib import Path

# 타입 정의 (CheckType - 보정 ④)
CheckType = Literal["hallucination", "omission", "contradiction", "math_error"]

# --- 중첩 모델들 (모두 extra="forbid" 추가) ---

class TargetAudience(BaseModel):
    model_config = ConfigDict(extra="forbid")
    
    level: str  # e.g. "대학 1학년"
    background_knowledge: list[str]
    exclude_knowledge: list[str]

class Expansion(BaseModel):
    model_config = ConfigDict(extra="forbid")
    
    max_depth: int = Field(default=4, ge=1, le=10)
    max_children_per_node: int = Field(default=5, ge=1, le=20)

class Dedup(BaseModel):
    model_config = ConfigDict(extra="forbid")
    
    similarity_threshold: float = Field(default=0.88, ge=0.0, le=1.0)
    embedding_model: str = Field(
        default="sentence-transformers/all-MiniLM-L6-v2"
    )

class Verification(BaseModel):
    model_config = ConfigDict(extra="forbid")
    
    max_retries: int = Field(default=1, ge=0, le=5)
    min_confidence: float = Field(default=0.7, ge=0.0, le=1.0)
    check_types: list[CheckType]  # 보정 ④: Literal 제약

class Claude(BaseModel):
    model_config = ConfigDict(extra="forbid")
    
    cli_path: str = Field(default="claude")
    mode: Literal["live", "cache", "dry_run"] = Field(default="cache")
    max_total_calls: int = Field(default=500, ge=1)
    timeout_seconds: int = Field(default=600, ge=1)
    sleep_between_calls: int = Field(default=3, ge=0)

class Paths(BaseModel):
    # 보정 ⑤: extra="forbid" 추가
    model_config = ConfigDict(extra="forbid")
    
    # 보정 ①: 모든 필드가 Path 타입 (str이 아님)
    # YAML의 문자열은 Pydantic이 자동으로 Path로 변환함
    pdf_input: Path      # e.g. Path("data/papers") → 절대 경로로 정규화
    cache_dir: Path      # e.g. Path("data/cache") → 절대 경로로 정규화
    output_dir: Path     # e.g. Path("data/output") → 절대 경로로 정규화
    checkpoints_dir: Path  # e.g. Path("checkpoints") → 절대 경로로 정규화

class Config(BaseModel):
    # 보정 ⑤: extra="forbid"
    model_config = ConfigDict(extra="forbid")
    
    target_audience: TargetAudience
    expansion: Expansion
    dedup: Dedup
    verification: Verification
    claude: Claude
    paths: Paths
```

## 4. 경로 처리 상세 로직 (보정 ①③ 통합)

### 데이터 플로우:

```
YAML 파일 (config.yaml)
    ↓
문자열: paths.pdf_input = "data/papers"
    ↓
Pydantic 검증 (자동 변환)
    ↓
경로: Path("data/papers")  ← 상대 경로로 아직 유지
    ↓
load_config() 함수 내에서 정규화
    ↓
절대 경로: Path("/absolute/path/to/project/data/papers")
    ↓
디렉토리 생성
    ↓
최종 Config 객체 반환
```

### load_config() 구현 의사코드:

```python
def load_config(path: str = "config.yaml") -> Config:
    # Step 1: config.yaml의 절대 경로와 프로젝트 루트 구하기 (보정 ③)
    config_path_abs = Path(path).resolve()
    project_root = config_path_abs.parent
    
    if not config_path_abs.exists():
        raise FileNotFoundError(f"config.yaml을 찾을 수 없습니다: {config_path_abs}")
    
    # Step 2: YAML 파싱
    with open(config_path_abs) as f:
        raw_data = yaml.safe_load(f)
    
    # Step 3: Pydantic 검증 (이때 Path 자동 변환 발생)
    #         하지만 경로는 아직 상대 경로 상태
    try:
        config = Config(**raw_data)
    except ValueError as e:
        raise ValueError(f"config.yaml 검증 실패: {e}")
    
    # Step 4: 경로 절대화 (프로젝트 루트 기준)
    config.paths.pdf_input = (project_root / config.paths.pdf_input).resolve()
    config.paths.cache_dir = (project_root / config.paths.cache_dir).resolve()
    config.paths.output_dir = (project_root / config.paths.output_dir).resolve()
    config.paths.checkpoints_dir = (project_root / config.paths.checkpoints_dir).resolve()
    
    # Step 5: 디렉토리 생성 (보정 ②: claude_responses/ 제외)
    #         각 모듈이 자기 책임 하에 하위 폴더 생성
    for path_field_name in ["pdf_input", "cache_dir", "output_dir", "checkpoints_dir"]:
        path_obj = getattr(config.paths, path_field_name)
        path_obj.mkdir(parents=True, exist_ok=True)
    
    return config
```

## 5. 검증 규칙 (수정판)

| 필드 | 타입 | 기본값 | 제약 조건 | 
|------|------|--------|---------|
| `expansion.max_depth` | int | 4 | 1 ≤ x ≤ 10 |
| `expansion.max_children_per_node` | int | 5 | 1 ≤ x ≤ 20 |
| `dedup.similarity_threshold` | float | 0.88 | 0.0 ≤ x ≤ 1.0 |
| `verification.min_confidence` | float | 0.7 | 0.0 ≤ x ≤ 1.0 |
| `verification.max_retries` | int | 1 | 0 ≤ x ≤ 5 |
| `verification.check_types` | list[CheckType] | ❌ 필수 | "hallucination", "omission", "contradiction", "math_error"만 허용 ← **보정 ④** |
| `claude.mode` | Literal | "cache" | "live"\|"cache"\|"dry_run" |
| `claude.max_total_calls` | int | 500 | x ≥ 1 |
| `claude.timeout_seconds` | int | 600 | x ≥ 1 |
| `claude.sleep_between_calls` | int | 3 | x ≥ 0 |

**경로 필드 (Paths 클래스)**:
| 필드 | 타입 | 설명 |
|------|------|------|
| `paths.pdf_input` | Path | 입력 PDF 디렉토리. config.yaml 기준 상대 경로 → 절대 경로로 정규화 |
| `paths.cache_dir` | Path | 캐시 루트. **하위 directory structure (e.g., claude_responses/)는 각 모듈이 생성** ← **보정 ②** |
| `paths.output_dir` | Path | 최종 산출물 디렉토리 |
| `paths.checkpoints_dir` | Path | 체크포인트 디렉토리 |

**전역 검증 (모든 모델)**:
- `extra="forbid"` ← **보정 ⑤**: 알 수 없는 필드 발견 시 즉시 ValidationError

## 6. 에러 처리 방식

| 상황 | 예외 타입 | 메시지 예시 |
|------|----------|----------|
| `config.yaml` 파일 없음 | `FileNotFoundError` | `"config.yaml을 찾을 수 없습니다: /path/to/config.yaml"` |
| YAML 파싱 실패 | `yaml.YAMLError` | (Pyyaml 기본 메시지) |
| 필드 타입 오류 | `pydantic.ValidationError` | `"validation error for Config..."` (Pydantic 기본) |
| 알 수 없는 필드 (오타) | `pydantic.ValidationError` | `"Extra inputs are not permitted..."` ← **보정 ⑤** |
| 범위 초과 (e.g., max_depth=11) | `pydantic.ValidationError` | `"Input should be less than or equal to 10..."` |
| check_types에 오타 | `pydantic.ValidationError` | `"Input should be 'hallucination', 'omission', 'contradiction' or 'math_error'..."` ← **보정 ④** |
| 디렉토리 생성 권한 부족 | `OSError` | (OS 기본 메시지) |

## 8. 아키텍처 특이사항

### ① 모듈 책임 경계 (보정 ②)

```
config.py (load_config)
├── data/papers/ ── 생성
├── data/cache/ ── 생성
│   └── claude_responses/ ── claude_client.py가 생성
├── data/output/ ── 생성
└── checkpoints/ ── 생성

이유: config.py는 설정 로드만 담당.
     내부 구조(예: cache 하위 폴더)는 각 모듈의 책임.
     config.py가 알면 모듈 간 의존성이 증가하고 수정이 어려워짐.
```

### ② 경로 기준점 (보정 ③)

```
❌ 나쁜 설계: CWD 기준
$ cd /tmp && python /project/src/main.py
→ 경로가 /tmp 기준으로 꼬임

✓ 좋은 설계: config.yaml 기준
config.yaml 위치 = 프로젝트 루트
$ cd /tmp && python /project/src/main.py
→ 모든 경로는 /project 기준으로 동작
```

### ③ Path 타입의 장점 (보정 ①)

```python
# 다른 모듈에서 사용할 때

# ❌ 나쁜 표현: 계속 변환 필요
pdf_path = Path(config["paths"]["pdf_input"]) / "paper.pdf"

# ✓ 좋은 표현: 바로 Path 연산
pdf_path = config.paths.pdf_input / "paper.pdf"
```

### ④ CheckType Literal (보정 ④)

```python
# ❌ 나쁜 표현: 오타 무시
check_types: list[str]  # 사용자가 "hallucnation"이라 써도 통과

# ✓ 좋은 표현: 오타 즉시 감지
CheckType = Literal["hallucination", "omission", "contradiction", "math_error"]
check_types: list[CheckType]  # Pydantic이 검증
```

### ⑤ extra="forbid" (보정 ⑤)

```yaml
# ❌ 오타가 조용히 무시됨 (기본 Pydantic)
expansion:
  max_dept: 4  # 오타! 하지만 기본값 4 사용, 사용자는 "내가 설정했다"고 착각

# ✓ 오타 즉시 감지 (extra="forbid")
expansion:
  max_dept: 4  # ValidationError: Extra inputs are not permitted
```

## 📋 결정 사항 정리

| 항목 | 결정 |
|------|------|
| 경로 자동 생성 위치 | `load_config()` 함수 내 (모델과 분리) |
| 하위 폴더 책임 | 각 모듈이 필요할 때 생성 (e.g., claude_responses/ → claude_client.py) |
| 경로 기준점 | config.yaml 파일이 있는 디렉토리 |
| Paths 필드 타입 | pathlib.Path (str이 아님) |
| 검증 엄격도 | extra="forbid" + Literal 타입으로 오타 방지 |
