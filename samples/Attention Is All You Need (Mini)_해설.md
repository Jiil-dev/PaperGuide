## 1. Abstract

이 문단은 2017년에 발표된 "Attention Is All You Need"라는 유명 논문의 **초록(논문 전체 내용을 요약한 첫 문단)**입니다. 핵심 내용을 풀어쓰면 다음과 같습니다.

**배경 — 기존 방식의 한계:**
당시 "한 언어의 문장을 다른 언어로 번역하기"처럼, 순서가 있는 데이터(sequence, 시퀀스)를 입력받아 다른 순서 있는 데이터로 변환(transduction)하는 작업에는 주로 두 종류의 신경망(neural network, 컴퓨터가 데이터를 학습하는 수학적 모델)이 쓰였습니다.
- **순환 신경망(Recurrent Neural Network)**: 단어를 하나씩 순서대로 처리하는 방식
- **합성곱 신경망(Convolutional Neural Network)**: 주변 단어들을 묶어서 한꺼번에 처리하는 방식

이 모델들은 보통 **인코더(encoder, 입력 문장을 압축·요약하는 부분)**와 **디코더(decoder, 요약된 정보를 바탕으로 출력 문장을 생성하는 부분)**로 구성됩니다. 그리고 성능이 좋은 모델은 인코더와 디코더를 **어텐션(attention, "지금 출력할 단어와 가장 관련 깊은 입력 단어가 어디인지" 집중하게 해 주는 기법)** 메커니즘으로 연결합니다.

**제안 — 트랜스포머(Transformer):**
이 논문의 저자들은 순환이나 합성곱을 **전혀 사용하지 않고**, 오직 어텐션 메커니즘**만**으로 이루어진 새로운 구조인 **트랜스포머(Transformer)**를 제안합니다.

**결과:**
기계 번역(machine translation, 컴퓨터가 자동으로 언어를 번역하는 작업) 실험 두 가지에서 트랜스포머는:
1. 번역 **품질**이 기존 모델보다 우수했고,
2. **병렬화(parallelizable, 여러 계산을 동시에 처리할 수 있는 성질)**가 더 쉬워서,
3. 학습에 필요한 **시간이 크게 줄었습니다**.

쉽게 비유하면, 기존 모델이 책을 한 글자씩 순서대로 읽는 방식이라면, 트랜스포머는 책 전체를 펼쳐놓고 중요한 부분에 형광펜을 치며 동시에 파악하는 방식에 가깝습니다.

### 1.1. 신경망(Neural Network) 기초

## 신경망(Neural Network)이란?

신경망은 **컴퓨터가 데이터를 입력받아 스스로 패턴을 학습하는 수학적 모델**입니다. 이름에서 짐작할 수 있듯이, 사람의 뇌에서 뉴런(신경세포)들이 서로 연결되어 정보를 처리하는 방식에서 아이디어를 빌려왔습니다.

### 핵심 비유: 여러 단계의 "판단 필터"

여러분이 사진을 보고 "이것은 고양이다"라고 판단하는 과정을 떠올려 보세요.
1. 먼저 눈으로 **점과 선**(밝기, 윤곽)을 봅니다.
2. 그 정보를 조합해 **귀 모양, 눈 모양** 같은 부분을 인식합니다.
3. 최종적으로 "고양이"라는 **결론**을 내립니다.

신경망도 이와 비슷하게, 데이터를 **여러 단계(층, layer)**에 걸쳐 변환하면서 점점 복잡한 패턴을 잡아냅니다.

### 기본 구조

신경망은 크게 세 부분으로 나뉩니다.

- **입력층(Input Layer)**: 데이터가 처음 들어오는 곳. 예를 들어 이미지의 각 픽셀 밝기값이 입력이 됩니다.
- **은닉층(Hidden Layer, 중간 처리 층)**: 입력을 수학적으로 변환하여 패턴을 추출하는 층. 여러 개가 쌓일 수 있습니다.
- **출력층(Output Layer)**: 최종 결과를 내보내는 곳. 예를 들어 "고양이일 확률 90%, 강아지일 확률 10%"처럼 결과를 출력합니다.

### 작동 원리 요약

각 층에서 일어나는 일을 수식으로 표현하면 다음과 같습니다.

$$y = f(w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b)$$

- $x_1, x_2, \ldots, x_n$: **입력값**들 (예: 픽셀 밝기)
- $w_1, w_2, \ldots, w_n$: **가중치(weight)** — 각 입력이 결과에 얼마나 중요한지를 나타내는 숫자. 이 숫자들이 바로 컴퓨터가 "학습"을 통해 조정하는 대상입니다.
- $b$: **편향(bias)** — 결과를 전체적으로 조절해주는 보정값 (일차함수 $y = ax + b$에서 $b$와 비슷한 역할)
- $f$: **활성화 함수(activation function)** — 계산 결과를 비선형적(단순 직선이 아닌 곡선 형태)으로 변환해 주는 함수. 이것이 없으면 아무리 층을 쌓아도 결국 단순한 직선 관계밖에 표현하지 못합니다.

### "학습"이란?

신경망의 학습이란, 정답과 신경망의 예측 사이의 **오차(틀린 정도)**를 계산한 뒤, 그 오차가 줄어드는 방향으로 가중치($w$)와 편향($b$)을 반복적으로 조금씩 수정하는 과정입니다. 고등학교 수학에서 함수의 최솟값을 찾기 위해 미분을 사용하는 것과 같은 원리입니다.

정리하면, 신경망은 **"입력 → 가중치를 곱하고 더하기 → 활성화 함수 통과"를 여러 층에 걸쳐 반복하며, 오차를 줄이는 방향으로 가중치를 스스로 조정하는 수학적 모델**입니다.

#### 1.1.1. 퍼셉트론(Perceptron)과 가중합

## 퍼셉트론(Perceptron)과 가중합

### 한 줄 요약
퍼셉트론은 **여러 개의 입력값에 각각 '중요도(가중치)'를 곱한 뒤, 그 결과를 모두 더해서 하나의 출력을 만들어내는 계산 장치**입니다. 이것이 인공 신경망(컴퓨터가 사람의 뇌를 흉내 내도록 만든 계산 구조)의 가장 작은 단위입니다.

---

### 일상적인 비유로 이해하기

여러분이 점심 메뉴를 고를 때를 생각해 보세요.

| 고려 요소 (입력) | 나에게 얼마나 중요한가 (가중치) | 오늘의 점수 (입력값) |
|---|---|---|
| 맛 | ×3 (매우 중요) | 8점 |
| 가격 | ×2 (꽤 중요) | 5점 |
| 거리 | ×1 (덜 중요) | 7점 |

최종 점수 = (8 × 3) + (5 × 2) + (7 × 1) = 24 + 10 + 7 = **41점**

이처럼 각 요소에 "얼마나 중요한지"를 곱해서 모두 더하는 것, 이것이 바로 **가중합(weighted sum, 가중치를 곱한 뒤 합산한 값)**입니다.

---

### 수식으로 표현하기

입력이 $x_1, x_2, \dots, x_n$이고, 각 입력에 대응하는 가중치(weight, 각 입력이 결과에 얼마나 영향을 미치는지를 나타내는 숫자)가 $w_1, w_2, \dots, w_n$일 때, 가중합은 다음과 같습니다:

$$z = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n$$

프로그래밍으로 표현하면 아래와 같습니다 (파이썬 스타일):

```
z = 0
for i in range(n):
    z += w[i] * x[i]
```

고등학교 수학에서 배운 **일차함수** $y = ax$의 확장판이라고 생각하면 됩니다. 입력이 1개이면 $y = wx$이고, 입력이 여러 개이면 각각에 서로 다른 기울기(가중치)를 곱해 모두 더하는 것입니다.

---

### 퍼셉트론의 동작 과정

1. **입력 받기**: 여러 개의 숫자 $x_1, x_2, \dots, x_n$이 들어옵니다.
2. **가중합 계산**: 각 입력에 가중치를 곱하고 모두 더합니다 → $z = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n$
3. **출력 결정**: 가중합 $z$가 일정 기준(문턱값, threshold)보다 크면 "1(예)"을, 작으면 "0(아니오)"을 출력합니다.

이 세 단계가 원문에서 말하는 "여러 입력에 가중치를 곱해 더한 뒤 결과를 출력하는 구조"에 해당합니다.

---

### 왜 '신경망의 최소 단위'인가?

사람의 뇌에는 뉴런(신경세포)이 있고, 뉴런 하나는 여러 신호를 받아 종합한 뒤 다음 뉴런에 신호를 보낼지 말지 결정합니다. 퍼셉트론은 바로 이 **뉴런 하나의 동작**을 수식으로 모방한 것입니다. 이 퍼셉트론을 수십~수백만 개 연결하면 복잡한 인공 신경망이 만들어집니다.

---

### 핵심 정리
- **가중치(weight)**: 각 입력의 중요도를 나타내는 숫자. 이 값을 조절하는 것이 곧 '학습'입니다.
- **가중합(weighted sum)**: 입력 × 가중치를 모두 더한 값.
- **퍼셉트론(Perceptron)**: 가중합을 계산하고, 그 결과로 출력을 내보내는 신경망의 최소 구성 요소.

#### 1.1.2. 활성화 함수(Activation Function)

## 활성화 함수(Activation Function)란?

신경망(인간의 뇌 신경세포를 흉내 낸 계산 구조)에서는 입력값들에 각각의 가중치(중요도를 나타내는 숫자)를 곱한 뒤 모두 더하는 과정을 거칩니다. 이렇게 구한 값을 **가중합**(weighted sum)이라 부릅니다.

예를 들어, 입력이 $x_1, x_2$이고 가중치가 $w_1, w_2$라면 가중합은 다음과 같습니다:

$$z = w_1 x_1 + w_2 x_2 + b$$

여기서 $b$는 편향(bias, 기준점을 조정하는 상수)입니다. 이 식은 고등학교에서 배운 **일차함수**(직선)의 형태와 같습니다.

### 왜 활성화 함수가 필요한가?

문제는, 직선(일차함수)만으로는 **곡선 형태의 복잡한 패턴**을 표현할 수 없다는 점입니다. 직선을 아무리 여러 개 더하거나 이어 붙여도 결과는 여전히 직선입니다. 이것은 마치 자(ruler)만으로 원이나 물결 모양을 그리려는 것과 같습니다.

**활성화 함수**는 이 가중합 결과 $z$를 **비선형(곡선 형태)**으로 변환해 주는 함수입니다. 즉, 직선적인 계산 결과를 굽혀서 곡선을 만들 수 있게 해 줍니다.

### 대표적인 예시

프로그래밍으로 쉽게 표현하면 다음과 같습니다:

- **ReLU**(Rectified Linear Unit): 0보다 작으면 0으로, 0 이상이면 그대로 내보내는 함수
  $$f(z) = \max(0,\, z)$$
  예: $f(-3) = 0$, $f(5) = 5$

이 함수는 단순해 보이지만, 0 이하를 꺾어 버리는 것만으로도 "비선형"이 됩니다. 고등학교에서 배운 그래프로 그려 보면, 왼쪽 절반은 $y=0$(x축에 붙어 있음)이고, 오른쪽 절반은 $y=z$(대각선)인 꺾인 모양입니다.

### 핵심 요약

| 활성화 함수가 없을 때 | 활성화 함수가 있을 때 |
|---|---|
| 가중합 → 직선(일차함수) 그대로 출력 | 가중합 → 곡선(비선형) 변환 후 출력 |
| 아무리 층을 쌓아도 직선 | 층을 쌓을수록 더 복잡한 곡선 표현 가능 |

정리하면, **활성화 함수는 가중합 결과를 비선형(곡선 형태)으로 변환하여 신경망이 복잡한 패턴을 표현할 수 있게 하는 함수**입니다. 직선만으로는 불가능한 복잡한 판단(예: 사진 속 고양이와 개 구별)을 가능하게 만드는 핵심 장치라고 이해하면 됩니다.

#### 1.1.3. 손실 함수(Loss Function)와 오차 측정

## 손실 함수(Loss Function)와 오차 측정

### 핵심 아이디어
신경망(입력을 받아 예측값을 내놓는 계산 구조)이 내놓은 **예측값**과 실제 **정답** 사이의 차이, 즉 "얼마나 틀렸는가"를 **하나의 숫자**로 요약해 주는 것이 손실 함수입니다.

### 왜 필요한가?
시험을 본 뒤 "몇 점인지" 점수를 매겨야 어디가 부족한지 알 수 있듯이, 신경망도 자신의 예측이 정답에서 얼마나 벗어났는지를 숫자 하나로 알아야 스스로를 개선(학습)할 수 있습니다. 이 숫자가 바로 **손실(loss)** 값이며, 손실 값을 계산하는 공식을 **손실 함수**라고 부릅니다.

### 가장 간단한 예: 평균 제곱 오차(MSE)
고등학교 수학에서 두 값의 차이를 구하는 것은 뺄셈입니다. 손실 함수도 이 아이디어에서 출발합니다.

1. **오차 구하기**: 예측값에서 정답을 뺍니다.
   - 예: 정답이 100인데 예측이 90이면, 오차 = 90 − 100 = −10

2. **부호 제거**: 오차가 양수든 음수든 "틀린 정도"는 같으므로, 제곱하여 부호를 없앱니다.
   - (−10)² = 100

3. **평균 내기**: 데이터가 여러 개일 때, 각각의 제곱 오차를 모두 더한 뒤 개수로 나눕니다.

수식으로 쓰면 다음과 같습니다:

$$\text{MSE} = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

여기서 $y_i$는 $i$번째 데이터의 정답, $\hat{y}_i$는 신경망이 내놓은 예측값, $n$은 데이터의 총 개수입니다. 이 값이 **0에 가까울수록** 예측이 정답에 가깝다는 뜻이고, **클수록** 많이 틀렸다는 뜻입니다.

### 일상적 비유
- 양궁 선수가 과녁 중심에서 얼마나 벗어났는지를 재는 것과 같습니다.
- 화살 10발을 쏜 뒤, 각 화살이 중심에서 떨어진 거리를 제곱해서 평균 내면 → 그것이 바로 이 선수의 "손실 값"입니다.
- 손실 값이 작을수록 실력이 좋은 것이고, 신경망은 이 값을 줄이는 방향으로 자기 내부의 숫자(가중치)를 조정합니다.

### 정리
| 요소 | 의미 |
|------|------|
| 예측값 ($\hat{y}$) | 신경망이 내놓은 답 |
| 정답 ($y$) | 실제로 맞는 값 |
| 손실(loss) | 예측값과 정답 사이의 차이를 하나의 숫자로 요약한 것 |
| 손실 함수 | 손실을 계산하는 공식 (예: 평균 제곱 오차) |

손실 함수가 내놓는 이 숫자 하나가 신경망 학습의 출발점입니다. 신경망은 이 숫자를 최대한 줄이는 것을 목표로 내부 값들을 반복적으로 수정해 나갑니다.

### 1.2. 순환 신경망(Recurrent Neural Network)

순환 신경망(Recurrent Neural Network, 줄여서 RNN)은 **순서가 있는 데이터**(예: 문장 속 단어들, 시간에 따른 주가 변화 등)를 처리하기 위해 설계된 특별한 신경망(neural network, 인간의 뇌 신경세포 연결을 흉내 낸 컴퓨터 계산 구조) 입니다.

## 핵심 아이디어: "이전 것을 기억한다"

일반적인 신경망은 입력 하나를 받으면 그것만 보고 결과를 내놓습니다. 하지만 우리가 문장을 읽을 때를 생각해 보세요. "나는 어제 학교에 갔다"라는 문장에서 "갔다"의 의미를 이해하려면 앞에 나온 "나는", "어제", "학교에"를 기억하고 있어야 합니다.

RNN은 바로 이 점을 모방합니다. 데이터를 **한 번에 하나씩 차례로** 입력받으면서, 매 단계마다 **숨겨진 상태(hidden state)**라는 일종의 '메모장'을 유지합니다. 이 메모장에는 이전 단계까지 처리한 정보가 요약되어 담겨 있습니다.

## 작동 방식을 비유로 이해하기

책을 한 페이지씩 읽는 사람을 상상해 보세요.
1. **1페이지**를 읽고 → 머릿속에 내용을 요약해 기억합니다.
2. **2페이지**를 읽을 때 → 1페이지의 기억 + 2페이지 내용을 함께 고려해 새로운 요약을 만듭니다.
3. **3페이지**를 읽을 때 → 앞의 요약 + 3페이지 내용으로 또 새로운 요약을 만듭니다.
4. 이 과정을 마지막 페이지까지 반복합니다.

RNN도 정확히 이 방식으로 동작합니다. 각 단계에서 **(현재 입력) + (이전까지의 기억)** → **(새로운 기억)** 을 계산하며, 이 "기억"이 바로 hidden state입니다.

## 왜 중요한가?

순서가 중요한 데이터 — 자연어(사람의 언어), 음성, 시계열 데이터(시간에 따라 변하는 데이터) — 를 다룰 때, 앞뒤 맥락을 고려할 수 있어 단순한 신경망보다 훨씬 더 적절한 결과를 냅니다. 번역, 음성 인식, 주가 예측 등 다양한 분야에서 기초가 되는 구조입니다.

#### 1.2.2. 가중치와 학습(Weight & Training)

## 가중치와 학습(Weight & Training)

### 가중치란 무엇인가?

신경망(뇌의 뉴런 연결을 흉내 낸 컴퓨터 모델)은 수많은 연결선으로 이루어져 있습니다. **가중치(weight)**란 이 연결선 하나하나에 붙어 있는 "중요도 숫자"입니다.

일상적인 비유로 설명하면, 여러분이 "오늘 우산을 가져갈까?"를 결정할 때 여러 정보를 참고한다고 합시다.

- 날씨 앱 예보 → 꽤 중요 → 가중치 **0.7**
- 친구의 "비 올 것 같아" 한마디 → 조금 참고 → 가중치 **0.2**
- 어제 날씨 → 별로 안 중요 → 가중치 **0.1**

이처럼 각 정보에 곱해지는 숫자가 가중치입니다. 가중치가 크면 해당 입력이 최종 판단에 더 큰 영향을 주고, 작으면 영향이 적습니다. 신경망에서는 이런 가중치가 수백만~수십억 개 존재합니다.

### 학습이란 무엇인가?

**학습(Training)**이란 신경망이 데이터를 반복적으로 보면서 가중치를 자동으로 조정하여, 정답에 더 가까운 출력을 내도록 만드는 과정입니다.

구체적인 흐름은 다음과 같습니다:

1. **초기화**: 처음에는 가중치를 아무렇게나(랜덤으로) 설정합니다. 당연히 이 상태에서는 엉뚱한 답을 냅니다.
2. **예측**: 데이터를 신경망에 넣으면, 각 연결선의 가중치가 곱해지면서 어떤 결과가 출력됩니다.
3. **오차 확인**: 출력된 결과와 실제 정답을 비교하여 "얼마나 틀렸는지"를 숫자로 계산합니다. (이 숫자를 '손실(loss)'이라 부릅니다.)
4. **가중치 조정**: 오차를 줄이는 방향으로 각 가중치를 아주 조금씩 수정합니다. 마치 라디오 주파수 다이얼을 미세하게 돌려서 소리가 더 선명해지도록 맞추는 것과 비슷합니다.
5. **반복**: 2~4번 과정을 수천~수만 번 반복하면, 가중치들이 점점 좋은 값으로 수렴하여 신경망의 정확도가 올라갑니다.

### 핵심 정리

| 용어 | 의미 |
|------|------|
| 가중치(weight) | 각 연결에 붙은 중요도 숫자. 입력값에 곱해져서 출력에 영향을 줌 |
| 학습(training) | 데이터를 통해 가중치를 자동 조정하여 정확도를 높이는 반복 과정 |

결국 **"신경망을 학습시킨다"는 것은 곧 "좋은 가중치 값을 찾는다"는 뜻**입니다. 프로그래머가 가중치를 하나하나 직접 정하는 것이 아니라, 컴퓨터가 수많은 데이터를 보며 스스로 최적의 값을 찾아가는 것이 핵심입니다.

#### 1.2.3. 숨겨진 상태(Hidden State)

## 숨겨진 상태(Hidden State)란?

RNN(순환 신경망, 데이터를 순서대로 하나씩 읽으며 학습하는 인공지능 모델)은 문장이나 시계열 데이터처럼 **순서가 있는 정보**를 다룹니다. 이때 RNN이 각 단계(step)에서 "지금까지 읽은 내용을 요약해서 기억해 두는 숫자 묶음"이 바로 **숨겨진 상태(Hidden State)**입니다.

### 쉽게 비유하기

책을 한 페이지씩 읽는다고 상상해 보세요.

- 1페이지를 읽으면, 머릿속에 "지금까지의 줄거리 요약"이 생깁니다.
- 2페이지를 읽으면, 1페이지까지의 요약 **+** 2페이지의 새로운 내용을 합쳐서 요약이 업데이트됩니다.
- 3페이지, 4페이지… 계속 같은 방식으로 요약이 갱신됩니다.

이 "머릿속 줄거리 요약"에 해당하는 것이 숨겨진 상태입니다.

### 구체적으로 어떤 형태인가?

숨겨진 상태는 **벡터(vector)**, 즉 여러 개의 숫자를 한 줄로 나열한 묶음입니다. 예를 들어 `[0.3, -1.2, 0.7, 0.1]`처럼 생긴 숫자 리스트라고 생각하면 됩니다. 각 숫자가 "지금까지 읽은 정보의 어떤 특징"을 수치로 표현하고 있습니다.

### 왜 "숨겨진(Hidden)"이라고 부르나?

RNN의 입력(예: 단어)과 출력(예: 예측 결과)은 우리가 직접 볼 수 있지만, 이 내부 기억 벡터는 모델 **안쪽**에서만 만들어지고 전달되기 때문에 외부에서 직접 관찰하기 어렵습니다. 그래서 "숨겨진" 상태라고 부릅니다.

### 핵심 정리

| 특징 | 설명 |
|------|------|
| **역할** | 이전 단계까지의 정보를 요약·저장 |
| **형태** | 숫자 여러 개로 이루어진 벡터(숫자 묶음) |
| **갱신 방식** | 매 단계마다 "이전 숨겨진 상태 + 현재 입력"을 결합하여 새로운 숨겨진 상태를 만듦 |
| **이름의 유래** | 모델 내부에서만 존재하여 외부에서 직접 보이지 않으므로 "숨겨진" 상태 |

즉, 숨겨진 상태는 RNN이 순서대로 데이터를 처리할 때 **"여태까지 무슨 일이 있었는지"를 압축해서 기억하는 내부 메모장** 같은 역할을 합니다.

### 1.3. 합성곱 신경망(Convolutional Neural Network)

합성곱 신경망(Convolutional Neural Network, 줄여서 CNN)은 **인접한 데이터를 묶어서 한꺼번에 처리하는 필터(일종의 작은 틀) 기반 신경망(컴퓨터가 데이터를 학습하는 구조) 구조**입니다.

비유로 설명하면, 여러분이 큰 사진에서 고양이를 찾는다고 상상해 보세요. 사진 전체를 한꺼번에 보는 대신, 작은 돋보기(=필터)를 사진 위에서 한 칸씩 옮겨가며 "여기에 귀가 있나?", "여기에 눈이 있나?" 하고 부분부분을 살펴봅니다. 이 돋보기가 바로 **필터(filter)**이고, 돋보기를 슬라이드하면서 인접한 픽셀(화소)들을 묶어 계산하는 연산이 **합성곱(convolution)**입니다.

작동 원리를 단계별로 보면:
1. **필터 슬라이딩**: 예를 들어 3×3 크기의 작은 숫자 격자(필터)를 이미지 위에 올려놓고, 겹치는 부분의 숫자끼리 곱한 뒤 모두 더합니다. 그런 다음 필터를 한 칸 옆으로 옮겨 같은 계산을 반복합니다.
2. **특징 추출**: 이 과정을 거치면 원래 이미지에서 '가장자리', '질감' 같은 **특징(feature, 데이터에서 의미 있는 패턴)**이 자동으로 뽑혀 나옵니다.
3. **여러 층 쌓기**: 필터를 여러 겹 쌓으면, 처음에는 단순한 선이나 모서리를 찾고, 깊은 층으로 갈수록 눈, 코, 얼굴 같은 복잡한 패턴을 인식할 수 있게 됩니다.

CNN이 강력한 이유는, 필터가 이미지의 **어디에 있든** 같은 패턴을 동일하게 감지할 수 있고(위치에 상관없이 고양이 귀를 찾을 수 있음), 이미지 전체를 한 번에 처리하는 것보다 계산량이 훨씬 적다는 점입니다.

#### 1.3.2. 합성곱 연산(Convolution)

## 합성곱 연산(Convolution)이란?

합성곱 연산은 **작은 필터(filter, 작은 숫자 격자판)**를 데이터 위에서 한 칸씩 밀어가며(슬라이드하며), 겹치는 부분의 값끼리 곱한 뒤 모두 더하는 수학 연산입니다.

---

### 구체적인 예시로 이해하기

5×5 크기의 숫자 격자(예: 흑백 이미지의 밝기 값)가 있고, 3×3 크기의 작은 필터가 있다고 상상해 봅시다.

**1단계 — 필터를 왼쪽 위 구석에 올려놓기**
데이터의 왼쪽 위 3×3 영역과 필터가 정확히 겹칩니다.

**2단계 — 겹치는 칸끼리 곱하기**
겹친 9개의 칸에서 (데이터 값) × (필터 값)을 각각 계산합니다.
예를 들어, 데이터 한 칸의 값이 2이고 필터 같은 위치의 값이 3이면, 그 칸의 결과는 2 × 3 = 6입니다.

**3단계 — 곱한 결과를 모두 합산하기**
9개의 곱셈 결과를 전부 더합니다. 이 하나의 합산값이 출력 격자의 한 칸이 됩니다.

**4단계 — 필터를 한 칸 오른쪽으로 밀기(슬라이드)**
필터를 오른쪽으로 한 칸 옮긴 뒤, 2~3단계를 반복합니다. 오른쪽 끝에 도달하면 한 줄 아래로 내려가 다시 왼쪽부터 시작합니다. 이 과정을 데이터 전체에 대해 반복하면 새로운 출력 격자가 완성됩니다.

---

### 숫자 예시

데이터(3×3 일부):
```
1  0  2
3  1  0
0  2  1
```

필터(3×3):
```
1  0  1
0  1  0
1  0  1
```

계산: (1×1) + (0×0) + (2×1) + (3×0) + (1×1) + (0×0) + (0×1) + (2×0) + (1×1)
= 1 + 0 + 2 + 0 + 1 + 0 + 0 + 0 + 1 = **5**

이 5라는 값이 출력의 해당 위치 한 칸에 기록됩니다.

---

### 왜 유용한가?

필터 안의 숫자 배치를 바꾸면, 같은 슬라이드 동작으로도 데이터에서 서로 다른 **특징(패턴)**을 뽑아낼 수 있습니다. 예를 들어 이미지에서 가로선, 세로선, 모서리 등을 감지하는 필터를 만들 수 있습니다. 핵심은 "곱하고 더하기"를 필터를 밀어가며 반복한다는 단순한 구조에 있습니다.

#### 1.3.3. 필터와 특징 추출(Filter & Feature Extraction)

## 필터와 특징 추출(Filter & Feature Extraction)

### 핵심 아이디어
여러분이 스마트폰 카메라로 고양이 사진을 찍었다고 상상해 보세요. 이 사진은 컴퓨터 내부에서 수많은 숫자들의 격자(2차원 표)로 저장됩니다. 예를 들어 흑백 사진이라면 각 칸(픽셀)에 0(검정)~255(흰색) 사이의 밝기 값이 들어 있습니다.

그런데 컴퓨터가 이 숫자들만 보고 "이건 고양이다"라고 판단하려면, 먼저 **선, 모서리, 곡선** 같은 의미 있는 패턴을 찾아내야 합니다. 이 역할을 하는 것이 바로 **필터(filter)**입니다.

### 필터란 무엇인가?
필터는 보통 3×3이나 5×5처럼 **아주 작은 크기의 숫자 격자**입니다. 예를 들어 세로선을 감지하는 3×3 필터는 다음과 같이 생겼습니다:

```
[-1  0  1]
[-1  0  1]
[-1  0  1]
```

이 작은 격자를 원본 이미지 위에 올려놓고, 겹치는 위치의 숫자끼리 곱한 뒤 모두 더합니다. 이것은 고등학교에서 배운 곱셈과 덧셈만으로 이루어지는 간단한 연산입니다. 그런 다음 필터를 한 칸씩 옆으로, 아래로 이동(슬라이딩)하면서 같은 계산을 반복합니다. 마치 돋보기를 이미지 위에서 조금씩 옮기며 살펴보는 것과 비슷합니다.

### 계산 예시
이미지의 어떤 3×3 영역이 아래와 같다고 합시다:

```
[10  10  50]
[10  10  50]
[10  10  50]
```

위의 세로선 감지 필터를 적용하면:
(-1×10) + (0×10) + (1×50) + (-1×10) + (0×10) + (1×50) + (-1×10) + (0×10) + (1×50)
= (-10 + 0 + 50) + (-10 + 0 + 50) + (-10 + 0 + 50)
= 40 + 40 + 40 = **120**

결과가 큰 양수(120)라는 것은, 이 영역에 **세로 방향의 밝기 변화(= 세로선)**가 존재한다는 뜻입니다. 반대로 밝기가 균일한 영역에서는 결과가 0에 가깝게 나옵니다.

### "학습 가능하다"는 것의 의미
전통적인 이미지 처리에서는 사람이 직접 필터 안의 숫자를 정해주었습니다. 하지만 원문에서 말하는 필터는 **학습 가능한(learnable)** 필터입니다. 즉, 필터 안의 숫자들이 처음에는 아무렇게나(랜덤하게) 정해지지만, 컴퓨터가 수많은 예시 이미지를 보면서 "어떤 숫자 조합이 고양이의 귀 윤곽을 잘 잡아내는가?"를 스스로 알아내고, 필터의 숫자를 조금씩 수정해 나갑니다. 이 과정이 바로 '학습'입니다.

### 특징 추출이란?
필터를 이미지 전체에 슬라이딩하며 계산하면, 원본 이미지와 비슷한 크기의 **새로운 숫자 격자**가 만들어집니다. 이 새 격자를 **특징 맵(feature map, 패턴이 어디에 얼마나 강하게 나타나는지를 보여주는 지도)**이라고 부릅니다.

- 하나의 필터는 하나의 패턴(예: 세로선)을 감지합니다.
- 여러 개의 필터를 동시에 사용하면, 세로선·가로선·대각선·곡선 등 다양한 패턴을 한꺼번에 뽑아낼 수 있습니다.

이렇게 원본 이미지에서 의미 있는 패턴 정보만 골라내는 전체 과정을 **특징 추출(Feature Extraction)**이라고 합니다.

### 요약
| 구성 요소 | 설명 |
|---|---|
| **필터** | 작은 숫자 격자 (예: 3×3). 안의 숫자는 학습으로 자동 결정됨 |
| **슬라이딩** | 필터를 이미지 위에서 한 칸씩 이동하며 곱셈-덧셈 반복 |
| **특징 맵** | 필터 적용 결과로 나온 새 숫자 격자. 패턴의 위치와 강도를 담고 있음 |
| **특징 추출** | 여러 필터로 선·모서리 등 의미 있는 패턴을 자동으로 찾아내는 전체 과정 |

---


## 2. Introduction

이 섹션은 "Attention Is All You Need" 논문의 서론으로, **Transformer(트랜스포머)**라는 새로운 모델 구조를 제안하게 된 배경과 동기를 설명합니다.

**핵심 흐름을 순서대로 정리하면 다음과 같습니다:**

1. **기존의 주류 방법**: 문장이나 시계열 데이터처럼 순서가 있는 데이터를 처리할 때, RNN(순환 신경망, 데이터를 한 단어씩 순서대로 처리하는 인공지능 모델)과 그 개량형인 LSTM, GRU가 가장 뛰어난 성능을 보여 왔습니다. 이들은 언어 모델링(다음에 올 단어를 예측하는 작업)이나 기계 번역(한 언어를 다른 언어로 자동 변환) 같은 문제에서 표준처럼 사용되었습니다.

2. **기존 방법의 근본적 한계**: RNN 계열 모델은 입력 문장의 단어를 **한 번에 하나씩, 순서대로** 처리합니다. 즉, 시점 $t$의 숨겨진 상태(hidden state, 모델이 내부적으로 기억하는 정보) $h_t$를 계산하려면 반드시 이전 시점의 상태 $h_{t-1}$이 먼저 계산되어야 합니다. 이것은 마치 줄 서 있는 사람이 앞 사람이 끝나야만 차례가 오는 것과 같아서, **병렬 처리(여러 계산을 동시에 수행하는 것)**가 불가능합니다. 문장이 길어질수록 이 문제는 더욱 심각해지며, 학습 속도가 크게 느려집니다.

3. **Attention 메커니즘의 등장**: 한편, Attention(어텐션, 입력 데이터에서 현재 중요한 부분에 집중하는 기법) 메커니즘이 등장하여, 문장 내 멀리 떨어진 단어 간의 관계도 거리에 상관없이 직접 파악할 수 있게 되었습니다. 하지만 대부분의 경우 어텐션은 RNN과 **함께** 사용되었기 때문에, 순차 처리라는 근본 한계는 여전히 남아 있었습니다.

4. **본 논문의 제안 — Transformer**: 저자들은 RNN(순환 구조)을 **완전히 제거**하고, 오직 Attention 메커니즘만으로 입력과 출력 사이의 전체적인 관계를 파악하는 새로운 모델 구조인 **Transformer**를 제안합니다. 순환 구조가 없으므로 병렬 처리가 훨씬 용이하며, 8개의 P100 GPU(고성능 그래픽 처리 장치)로 단 12시간 학습만으로도 당시 최고 수준의 번역 품질을 달성했다고 보고합니다.

**요약**: "순서대로 하나씩 처리하느라 느렸던 기존 방식을 버리고, '어디가 중요한지'를 한꺼번에 파악하는 방식만으로 더 빠르고 더 좋은 결과를 얻겠다"는 것이 이 논문의 핵심 아이디어입니다.

### 2.2. 순환 신경망(RNN)과 순차 처리

## 순환 신경망(RNN)과 순차 처리

### 핵심 아이디어: "한 단어씩 순서대로 읽는 기계"

여러분이 영어 문장을 왼쪽에서 오른쪽으로 한 단어씩 읽는다고 상상해 보세요. "I love cats"라는 문장을 읽을 때, "I"를 먼저 읽고, 그 기억을 가진 채로 "love"를 읽고, 다시 그 기억을 가진 채로 "cats"를 읽습니다. 순환 신경망(RNN, Recurrent Neural Network — 이전 단계의 출력을 다시 자기 자신의 입력으로 되돌려 보내는 구조의 인공지능 모델)이 바로 이 방식으로 작동합니다.

### 작동 원리

RNN은 각 시간 단계(time step, 단어를 하나 처리하는 한 차례)마다 두 가지를 입력으로 받습니다:

1. **현재 단어** — 지금 읽고 있는 단어
2. **이전 단계의 숨겨진 상태** $h_{t-1}$ (hidden state — 지금까지 읽은 내용을 압축 요약한 숫자 묶음)

이 두 가지를 조합하여 **현재 단계의 숨겨진 상태** $h_t$를 만들어 냅니다. 이 $h_t$는 다시 다음 단계로 전달되어 $h_{t+1}$을 만드는 데 사용됩니다.

쉽게 비유하면, 친구에게 긴 이야기를 전화로 한 문장씩 전달하는 "전화 릴레이" 게임과 같습니다. 각 사람($h_t$)은 이전 사람($h_{t-1}$)이 전해준 요약과 새로 들은 문장을 합쳐서, 다음 사람에게 다시 요약을 전달합니다.

### 병렬화 불가 문제

여기서 치명적인 단점이 생깁니다. $h_t$를 계산하려면 **반드시** $h_{t-1}$이 먼저 완성되어야 합니다. 즉, 두 번째 단어를 처리하려면 첫 번째 단어의 처리가 끝나야 하고, 세 번째 단어를 처리하려면 두 번째 단어의 처리가 끝나야 합니다.

이것은 마치 공장 조립 라인에서 한 명의 작업자가 모든 부품을 순서대로 조립하는 것과 같습니다. 작업자가 10명 있어도 앞 작업이 끝나야 다음 작업을 시작할 수 있으므로, 여러 명이 동시에(병렬로) 일할 수 없습니다.

컴퓨터에서 **병렬화**(parallelization — 여러 계산을 동시에 수행하여 전체 처리 속도를 높이는 기법)가 불가능하다는 것은, GPU(Graphics Processing Unit — 수천 개의 작은 계산기를 동시에 돌릴 수 있는 하드웨어)의 강점을 제대로 활용할 수 없다는 뜻입니다. 문장이 길어질수록 처리 시간이 단어 수에 비례하여 늘어나, 학습과 추론 모두 매우 느려집니다.

이 병렬화 불가 문제가 이후 **Transformer**(트랜스포머)라는 새로운 구조가 등장하게 된 핵심 동기 중 하나입니다.

#### 2.2.2. 숨겨진 상태(Hidden State)와 시간 단계(Time Step)

## 숨겨진 상태(Hidden State)와 시간 단계(Time Step)

### 비유로 시작하기: 독서 노트

여러분이 긴 소설을 읽는다고 상상해 보세요. 한 페이지를 읽을 때마다, 머릿속에는 "지금까지 읽은 내용의 요약"이 자동으로 만들어집니다. 예를 들어 3페이지까지 읽었다면, "주인공이 집을 떠나 숲에 들어갔고, 이상한 노인을 만났다"라는 요약이 머릿속에 있겠지요. 4페이지를 읽을 때는 이 요약 + 4페이지의 새 내용을 합쳐서, 머릿속 요약이 업데이트됩니다.

RNN(순환 신경망, 순서가 있는 데이터를 처리하는 인공지능 모델)에서의 **숨겨진 상태(hidden state)**가 바로 이 "머릿속 요약 노트"에 해당합니다.

---

### 시간 단계(Time Step)란?

순서가 있는 데이터를 하나씩 처리하는 각각의 "차례"를 **시간 단계(time step)**라고 부릅니다.

예를 들어 "나는 학교에 간다"라는 문장을 단어 단위로 처리한다면:
- 시간 단계 1 (t=1): "나는"
- 시간 단계 2 (t=2): "학교에"
- 시간 단계 3 (t=3): "간다"

꼭 문장이 아니어도 됩니다. 주식 가격이 날짜별로 나열된 데이터라면, 1일차가 t=1, 2일차가 t=2, … 이런 식입니다. 핵심은 **데이터에 순서가 있고, 그 순서를 하나씩 밟아 나간다**는 것입니다.

---

### 숨겨진 상태($h_t$)란?

각 시간 단계 $t$에서 RNN은 **숨겨진 상태** $h_t$를 만들어냅니다. $h_t$는 **숫자 여러 개가 묶인 목록**(예: [0.3, -1.2, 0.7, …])으로, "시간 단계 1부터 $t$까지 입력된 모든 정보를 압축 요약한 것"입니다.

- $h_1$: 첫 번째 입력만 반영한 요약
- $h_2$: 첫 번째 + 두 번째 입력을 반영한 요약
- $h_3$: 첫 번째 + 두 번째 + 세 번째 입력을 반영한 요약

이 숫자 묶음 안의 각 숫자가 정확히 무엇을 뜻하는지는 사람이 직접 정하는 것이 아니라, 모델이 학습(훈련)을 통해 스스로 결정합니다. 그래서 "숨겨진(hidden)" 상태라고 부릅니다 — 사람이 직접 들여다봐도 각 숫자의 의미를 바로 알기 어렵기 때문입니다.

---

### 어떻게 다음 단계로 전달되는가?

RNN의 핵심 작동 방식을 한 줄로 요약하면 이렇습니다:

$$h_t = f(h_{t-1},\; x_t)$$

여기서:
- $x_t$: 시간 단계 $t$에서 새로 들어오는 입력 (예: 현재 단어)
- $h_{t-1}$: 바로 직전 시간 단계의 숨겨진 상태 (이전까지의 요약)
- $f$: 이 둘을 합쳐서 새로운 요약을 만드는 계산 규칙 (함수)
- $h_t$: 새롭게 업데이트된 숨겨진 상태

즉, **"이전까지의 요약($h_{t-1}$)"과 "지금 새로 들어온 정보($x_t$)"를 합쳐서 "새 요약($h_t$)"을 만드는 과정**이 매 시간 단계마다 반복됩니다.

가장 처음(t=1)에는 이전 요약이 없으므로, $h_0$을 보통 모든 값이 0인 숫자 묶음([0, 0, 0, …])으로 설정합니다.

---

### 구체적 예시: "나는 학교에 간다"

| 시간 단계 | 입력($x_t$) | 이전 요약($h_{t-1}$) | 새 요약($h_t$) |
|---|---|---|---|
| t=0 | (없음) | (없음) | $h_0$ = [0, 0, 0, …] (초기값) |
| t=1 | "나는" | $h_0$ | $h_1$ = $f(h_0, \text{"나는"})$ → "누군가가 주어" 정보 압축 |
| t=2 | "학교에" | $h_1$ | $h_2$ = $f(h_1, \text{"학교에"})$ → "누군가가 학교와 관련" 정보 압축 |
| t=3 | "간다" | $h_2$ | $h_3$ = $f(h_2, \text{"간다"})$ → "누군가가 학교에 가는 행위" 정보 압축 |

마지막 $h_3$에는 문장 전체의 의미가 숫자 묶음으로 압축되어 들어 있습니다.

---

### 핵심 정리

1. **시간 단계(time step)**: 순서 데이터를 하나씩 처리하는 각각의 차례 ($t = 1, 2, 3, \ldots$)
2. **숨겨진 상태($h_t$)**: 시간 단계 $t$까지의 모든 입력 정보를 압축 요약한 숫자 묶음
3. **전달 방식**: 매 단계에서 $h_t = f(h_{t-1}, x_t)$, 즉 "직전 요약 + 새 입력 → 새 요약"을 반복하며, 이를 통해 과거 정보가 미래 단계로 이어집니다.

#### 2.2.3. 병렬 처리와 GPU의 역할

## 병렬 처리와 GPU의 역할

### 1. 병렬 처리란 무엇인가?

여러분이 프로그래밍 수업에서 작성하는 코드는 대부분 **순차 처리(명령을 한 줄씩 차례대로 실행하는 방식)**입니다. 예를 들어, 1000개의 숫자에 각각 2를 곱해야 한다면, 첫 번째 숫자에 2를 곱하고 → 두 번째 숫자에 2를 곱하고 → … → 1000번째까지 하나씩 반복합니다.

**병렬 처리(여러 계산을 동시에 수행하는 방식)**는 이와 다릅니다. 만약 일꾼이 1000명 있다면, 각 일꾼이 숫자 하나씩 맡아 동시에 2를 곱할 수 있습니다. 이렇게 하면 1000번 걸릴 작업이 사실상 1번의 시간 안에 끝납니다.

비유하자면:
- **순차 처리** = 계산대가 1개인 마트. 손님 100명이 한 줄로 서서 차례로 계산합니다.
- **병렬 처리** = 계산대가 100개인 마트. 손님 100명이 각자 계산대에서 동시에 계산합니다.

### 2. CPU vs GPU — 왜 구조가 다를까?

컴퓨터에는 두 종류의 주요 연산 장치가 있습니다.

**CPU(Central Processing Unit, 중앙 처리 장치)**는 컴퓨터의 "두뇌"입니다.
- 보통 4~16개 정도의 **코어(독립적으로 계산을 수행하는 단위)**를 가집니다.
- 각 코어가 매우 똑똑하고 빠릅니다. 복잡한 판단(if문 분기, 운영체제 관리 등)을 잘 처리합니다.
- 비유: **소수의 박사급 전문가**. 어려운 문제를 빠르게 풀지만, 인원이 적습니다.

**GPU(Graphics Processing Unit, 그래픽 처리 장치)**는 원래 화면에 그래픽을 그리기 위해 만들어진 장치입니다.
- 수천 개(보통 수백~수만 개)의 아주 작은 코어를 가집니다.
- 각 코어는 CPU 코어보다 단순하고 느리지만, **같은 작업을 동시에 수행하는 데** 최적화되어 있습니다.
- 비유: **수천 명의 단순 계산 알바생**. 한 사람이 풀 수 있는 문제는 단순하지만, 동시에 수천 명이 일하므로 전체 작업량이 엄청납니다.

### 3. 왜 GPU가 동시 계산에 강한가?

화면에 그래픽을 그리는 작업을 생각해 봅시다. 1920×1080 해상도의 화면은 약 200만 개의 점(픽셀)으로 이루어져 있습니다. 각 픽셀의 색상을 계산하는 작업은 서로 독립적이고, 계산 방법도 거의 동일합니다. 즉, "같은 종류의 단순한 계산을 수백만 번 반복"하는 것입니다.

GPU는 바로 이런 상황을 위해 설계되었습니다:
- **같은 명령을 수천 개의 데이터에 동시에 적용**하는 구조 (이를 SIMD — Single Instruction, Multiple Data, 즉 "하나의 명령으로 여러 데이터를 동시에 처리"라고 부릅니다)
- 수천 개의 작은 코어가 한꺼번에 동작하여 대량의 단순 계산을 빠르게 끝냅니다.

### 4. AI와 GPU의 관계

AI 모델(특히 딥러닝)에서는 거대한 숫자 배열(행렬)끼리의 곱셈과 덧셈이 핵심 연산입니다. 예를 들어, 수백만 개의 숫자를 각각 곱하고 더하는 작업이 반복됩니다. 이 작업들은:
- 서로 **독립적**이어서 동시에 계산할 수 있고,
- 모두 **같은 종류의 단순 연산**(곱셈, 덧셈)입니다.

이것은 GPU가 가장 잘하는 종류의 작업과 정확히 일치합니다. 그래서 AI 학습과 실행에 GPU가 핵심 하드웨어로 사용됩니다.

### 정리

| 항목 | CPU | GPU |
|------|-----|-----|
| 코어 수 | 적음 (4~16개) | 많음 (수천~수만 개) |
| 코어당 능력 | 복잡한 작업 가능 | 단순한 작업에 특화 |
| 적합한 작업 | 복잡한 분기·판단 | 같은 계산을 대량으로 동시 수행 |
| 비유 | 박사 몇 명 | 알바생 수천 명 |

핵심 요약: **병렬 처리**는 여러 계산을 동시에 수행하는 방식이며, **GPU**는 수천 개의 작은 코어를 통해 같은 종류의 단순 계산을 동시에 대량 처리하도록 설계된 하드웨어입니다. AI에서 필요한 대규모 숫자 연산이 바로 이 구조에 딱 맞기 때문에, GPU가 AI의 핵심 장비로 쓰입니다.

### 2.3. LSTM과 GRU

## LSTM과 GRU — 기본 RNN의 '건망증'을 고친 개량형 순환 신경망

### 왜 필요한가?

기본 RNN(순환 신경망, 이전 단계의 출력을 다음 단계의 입력으로 되먹이며 순서가 있는 데이터를 처리하는 인공 신경망)은 문장이 길어지면 앞쪽에 나온 중요한 정보를 점점 잊어버리는 치명적인 문제가 있습니다. 예를 들어 "나는 **프랑스**에서 10년간 살았고, 여러 도시를 여행했으며, 다양한 음식을 먹어 봤기 때문에 ___어를 잘한다"라는 긴 문장에서, 빈칸에 '프랑스'라는 단서를 기억해야 하지만 기본 RNN은 중간 내용이 길어질수록 그 단서를 잃어버립니다.

이 현상의 근본 원인은 **기울기 소실 문제**(학습 과정에서 오차 신호가 여러 단계를 거슬러 올라갈수록 점점 0에 가까워져 사실상 학습이 멈추는 현상)입니다.

### LSTM (Long Short-Term Memory, 장단기 기억 네트워크)

LSTM은 이 건망증 문제를 **세 개의 게이트(gate, 정보를 얼마나 통과시킬지 0~1 사이 값으로 조절하는 문)**로 해결합니다.

1. **망각 게이트(Forget Gate)**: "지금까지 기억하고 있던 정보 중 어떤 부분을 버릴까?"를 결정합니다.
2. **입력 게이트(Input Gate)**: "새로 들어온 정보 중 어떤 부분을 기억에 추가할까?"를 결정합니다.
3. **출력 게이트(Output Gate)**: "현재 기억 중 어떤 부분을 밖으로 내보낼까?"를 결정합니다.

이 세 게이트 덕분에 LSTM은 중요한 정보를 오래 보존하고, 불필요한 정보는 적절히 잊을 수 있습니다. 마치 공부할 때 형광펜으로 중요 부분만 표시하고, 불필요한 내용은 넘기며, 시험 답안에 필요한 것만 꺼내 쓰는 것과 비슷합니다.

### GRU (Gated Recurrent Unit, 게이트 순환 유닛)

GRU는 LSTM의 아이디어를 더 간결하게 만든 변형입니다. 세 개의 게이트 대신 **두 개의 게이트**만 사용합니다.

1. **리셋 게이트(Reset Gate)**: "과거 기억을 얼마나 무시할까?"를 결정합니다.
2. **업데이트 게이트(Update Gate)**: "과거 기억과 새 정보를 어떤 비율로 섞을까?"를 결정합니다. (LSTM의 망각 게이트와 입력 게이트 역할을 하나로 합친 것)

게이트 수가 적으므로 계산이 더 빠르고, 학습에 필요한 값(파라미터)이 더 적습니다. 성능은 많은 경우 LSTM과 비슷하지만, 데이터가 매우 복잡하고 긴 경우에는 LSTM이 유리할 수 있습니다.

### 핵심 요약

| 구분 | 기본 RNN | LSTM | GRU |
|------|---------|------|-----|
| 긴 문장 기억 | ✗ (잊어버림) | ✓ | ✓ |
| 게이트 수 | 없음 | 3개 | 2개 |
| 계산 속도 | 빠름 | 느림 | 중간 |

두 모델 모두 "기본 RNN이 긴 문장에서 앞쪽 정보를 잊어버리는 문제"를 게이트라는 장치를 도입해 해결한 개량형 순환 신경망입니다.

#### 2.3.1. RNN (순환 신경망) 기초

## RNN (Recurrent Neural Network, 순환 신경망) 기초

### 한 줄 요약
RNN은 **이전 단계의 출력을 다음 단계의 입력으로 되먹이는(feedback) 구조**를 가진 신경망(neural network, 뇌의 뉴런처럼 연결된 계산 단위들의 네트워크)으로, **순서가 있는 데이터**를 처리하는 데 특화되어 있습니다.

---

### "순서가 있는 데이터"란?

우리 주변의 많은 데이터는 **순서(sequence)**가 중요합니다.

- **문장**: "나는 학교에 간다"에서 단어 순서를 바꾸면 의미가 달라집니다.
- **시계열(time series, 시간 순서대로 나열된 숫자들)**: 주식 가격, 기온 변화 등은 어제 값이 오늘 값에 영향을 줍니다.
- **음악**: 음표의 순서가 멜로디를 결정합니다.

이런 데이터를 처리하려면, 컴퓨터가 **"이전에 무엇을 봤는지"를 기억**할 수 있어야 합니다.

---

### 되먹임(Recurrence)이란?

프로그래밍에서 for문을 떠올려 보세요. 반복문 안에서 이전 반복의 결과를 변수에 저장해 두고, 다음 반복에서 그 변수를 사용하는 것과 비슷합니다.

```python
# 비유: RNN의 핵심 아이디어를 단순 코드로 표현
memory = 0  # 기억(hidden state)을 0으로 초기화
for word in sentence:
    memory = process(memory, word)  # 이전 기억 + 현재 입력 → 새 기억
```

여기서 `memory`가 바로 RNN의 핵심인 **hidden state(은닉 상태, 신경망이 내부적으로 유지하는 '기억'에 해당하는 숫자 묶음)**입니다.

---

### RNN의 작동 방식 (단계별)

문장 "나는 학교에 간다"를 처리한다고 가정합시다.

| 시간 단계 | 현재 입력 | 이전 기억 | → | 새로운 기억 (출력) |
|:---------:|:---------:|:---------:|:-:|:------------------:|
| t=1 | "나는" | 없음 (초기값) | → | 기억₁ |
| t=2 | "학교에" | 기억₁ | → | 기억₂ |
| t=3 | "간다" | 기억₂ | → | 기억₃ |

각 단계에서 RNN은 다음 두 가지를 동시에 받아들입니다:
1. **현재 입력** (지금 들어온 단어)
2. **이전 단계의 기억** (지금까지의 문맥 정보)

이 둘을 합쳐 계산한 결과가 **새로운 기억**이 되고, 이것이 다시 다음 단계로 전달됩니다. 이것이 바로 "되먹임(recurrence)"입니다.

---

### 왜 "순환(循環)"이라고 부를까?

일반적인 신경망은 정보가 입력 → 출력 방향으로 **한 번만** 흐릅니다 (앞으로만 가는 일방통행 도로처럼). 그런데 RNN은 출력의 일부가 **다시 자기 자신에게 돌아옵니다**(루프를 그리며 순환). 이 순환 고리 덕분에 과거 정보를 기억할 수 있는 것입니다.

```
    ┌──────────┐
    │          ↓
입력 → [RNN 셀] → 출력
         ↑    │
         └────┘  ← 이전 출력이 다시 입력으로 되먹임
```

---

### 일상적 비유

RNN은 마치 **소설을 읽는 사람**과 같습니다. 10페이지를 읽을 때, 1~9페이지의 내용을 **머릿속에 요약·기억**하고 있기 때문에 현재 문장의 의미를 이해할 수 있습니다. 매 페이지마다 "지금까지의 요약(기억) + 새 페이지 내용"을 합쳐 "업데이트된 요약"을 만드는 셈입니다.

---

### 정리

- **핵심 아이디어**: 이전 단계의 출력(기억)을 다음 단계의 입력으로 되먹여서, 순서 정보를 활용한다.
- **용도**: 문장, 시계열(시간에 따라 변하는 데이터) 등 순서가 중요한 데이터를 처리한다.
- **구조적 특징**: 같은 계산 단위(RNN 셀)가 시간 단계마다 반복 사용되며, 매번 이전의 기억을 전달받는다.

#### 2.3.2. 기울기 소실 문제 (Vanishing Gradient Problem)

## 기울기 소실 문제 (Vanishing Gradient Problem)

이 개념을 이해하기 위해, 먼저 간단한 비유부터 시작하겠습니다.

### 전화 게임 비유

여러분이 20명이 일렬로 서서 첫 번째 사람의 메시지를 마지막 사람에게 전달하는 "전화 게임"을 한다고 상상해 보세요. 사람을 한 명 거칠 때마다 메시지가 조금씩 흐려지고, 20번째 사람에게 도달할 때쯤이면 원래 메시지가 거의 사라집니다. **기울기 소실 문제**가 바로 이것과 같은 현상입니다.

### 무엇이 소실되는가?

인공 신경망(뇌의 신경세포를 흉내 낸 컴퓨터 계산 구조)은 정답과 자신의 예측 사이의 차이, 즉 **오차 신호**를 이용해 학습합니다. 학습이란, 이 오차를 줄이기 위해 내부의 숫자(가중치)를 조금씩 조정하는 과정입니다.

이때 "얼마나, 어느 방향으로 조정해야 하는가"를 알려주는 값이 바로 **기울기(gradient)**입니다. 고등학교에서 배운 미분의 도함수 값, 즉 함수의 기울기와 같은 개념입니다. 기울기가 크면 "많이 고쳐라", 기울기가 작으면 "조금만 고쳐라"라는 뜻입니다.

### 왜 소실되는가?

신경망이 여러 층(단계)으로 쌓여 있을 때, 오차 신호는 출력(마지막 층)에서 입력(첫 번째 층) 방향으로 거꾸로 전달됩니다. 이 과정을 **역전파(오차를 뒤쪽으로 되돌려 보내는 계산 방법)**라고 합니다.

문제는, 각 층을 거칠 때마다 기울기 값에 **1보다 작은 수가 반복적으로 곱해진다**는 점입니다. 예를 들어 매 단계마다 0.3이 곱해진다면:

- 1단계 후: 0.3
- 2단계 후: 0.3 × 0.3 = 0.09
- 5단계 후: 0.3⁵ = 0.00243
- 10단계 후: 0.3¹⁰ ≈ 0.0000059

이처럼 1보다 작은 수를 여러 번 곱하면 값이 급격히 0에 가까워집니다. 이것이 원문에서 말하는 **"오차 신호가 여러 단계를 거슬러 올라갈수록 0에 가까워지는"** 현상입니다.

### 결과: 먼 과거의 정보를 학습하지 못함

기울기가 거의 0이 되면, "거의 고칠 필요 없다"는 잘못된 신호를 보내게 됩니다. 실제로는 크게 고쳐야 하는데도 말입니다. 결국 입력에 가까운 앞쪽 층(또는 시간 순서로 먼 과거의 데이터)은 사실상 학습이 멈추게 됩니다. 이것이 바로 **"먼 과거의 정보를 학습하지 못하게 되는"** 문제입니다.

### 정리

| 핵심 요소 | 설명 |
|---|---|
| **기울기** | 오차를 줄이기 위해 값을 얼마나 조정할지 알려주는 미분값 |
| **소실 원인** | 1보다 작은 수가 층마다 반복 곱해져 값이 급격히 0에 수렴 |
| **결과** | 앞쪽 층(또는 먼 과거)의 학습이 사실상 중단됨 |

#### 2.3.3. 게이트 메커니즘 (Gate Mechanism)

## 게이트 메커니즘 (Gate Mechanism)

게이트 메커니즘은 **정보의 흐름을 선택적으로 열고 닫는 구조**입니다. 이름 그대로 "문(gate)"처럼 작동한다고 생각하면 됩니다.

### 핵심 원리

1. **sigmoid 함수(시그모이드 함수)로 0~1 사이의 값을 만듭니다.**
   - sigmoid 함수란, 어떤 숫자를 넣으면 항상 0과 1 사이의 값으로 바꿔주는 함수입니다.
   - 수식으로는 $\sigma(x) = \frac{1}{1 + e^{-x}}$ 로 표현됩니다.
   - $x$가 매우 크면 결과는 1에 가까워지고, $x$가 매우 작으면(큰 음수이면) 결과는 0에 가까워집니다.

2. **이 0~1 값을 "문의 열림 정도"로 사용합니다.**
   - 값이 **1에 가까우면** → 문이 **활짝 열림** → 정보가 그대로 통과합니다.
   - 값이 **0에 가까우면** → 문이 **닫힘** → 정보가 차단됩니다.
   - 값이 **0.5 정도이면** → 문이 **반쯤 열림** → 정보의 절반 정도만 통과합니다.

### 비유로 이해하기

수도꼭지를 떠올려 보세요.
- 수도꼭지를 완전히 열면(=1) 물(정보)이 최대로 흐릅니다.
- 수도꼭지를 완전히 잠그면(=0) 물이 전혀 흐르지 않습니다.
- 중간쯤 열면(=0.3, 0.7 등) 그만큼만 물이 흐릅니다.

게이트 메커니즘은 이처럼 sigmoid 함수가 만들어낸 0~1 값을 원래 정보에 **곱하는 방식**으로 작동합니다. 예를 들어, 어떤 정보의 값이 5이고 게이트 값이 0.2라면, 실제로 통과하는 정보는 $5 \times 0.2 = 1$이 됩니다.

### 왜 필요한가?

AI 모델이 데이터를 처리할 때, 모든 정보가 동등하게 중요하지는 않습니다. 게이트 메커니즘을 통해 **"이 정보는 중요하니 통과시키고, 저 정보는 덜 중요하니 줄이거나 차단하자"**라는 판단을 자동으로 수행할 수 있습니다. 이때 어떤 정보를 얼마나 통과시킬지는 모델이 학습(데이터를 반복적으로 보며 스스로 조정하는 과정)을 통해 자동으로 결정합니다.
